CSS와 박스 모델

1. 박스 모델 
    - 실제 컨텐츠 영역, 패딩(padding), 박스의 테두리(border), 마진(margin) 등의 요소로 구성된다.
    - 개발자 도구 창에서 박스 모델 확인이 가능함.

    (1) display 속성 
        - block
        - inline
        - inline-block
        - none 

    (2) 테두리 관련 속성들 
        - border-width : 테두리 두께
        - border-color : 테두리 색
        - border-style : 위의 속성들을 한줄로 작성
        - border-radius : 박스 모서리 부분을 둥글게 처리    
        - box-shadow : 선택한 요소에 그림자 효과 내기





여백을 조절하는 속성들

1. margin 속성
    - 현재 요소 주변의 여백
    - 마진을 이용하면 요소와 요소 간의 간격 조절 가능함
    - margin : 0 auto;
    
    (1) 마진 중첩 현상
        - 요소를 세로로 배치할 경우, 마진과 마진이 만날 때 값이 더 큰 마진으로 겹쳐지는 것
        - 요소를 가로로 배치할 경우에는 해당하지 않음.

2. padding 속성
    - 컨텐츠 영역과 테두리 사이의 여백 (=테두리 안쪽 여백)





문서 구조를 위한 HTML5 시맨틱 태그

1. <header></header> 태그 : 머리말 지정
    - 사이트 전체의 제목 부분이 될 수도 있고, 본문의 제목 부분이 될 수도 있다.
    - 주로 페이지 맨 위쪽이나 왼쪽에 삽입
    - <form></form> 태그를 사용해 검색 창을 넣거나 <nav></nav> 태그를 이용해 사이트 메뉴를 넣음.

2. <nav></nav> 태그 : 네비게이션 링크
    - 같은 사이트 안의 문서나 다른 사이트의 문서로 연결하는 링크를 나타냄
    - 내비게이션 메뉴뿐만 아니라 footer에 있는 사이트 링크 모음 부분에도 사용됨.

3. <section></section> 태그 : 주제별 컨텐츠 영역
    - 문서에서 주제별로 컨텐츠를 묶을 때 사용
    - 섹션 제목을 나타내는 <h숫자></h숫자> 태그가 함께 사용됨

4. <article></article> 태그 : 컨텐츠 내용
    - 웹 페이지의 실제 내용
    - 태그 적용 부분을 떼어내 독립적으로 배포하거나 재사용하더라도 완전히 하나의 컨텐츠이다.

5. <aside></aside> 태그 : 본문 이외의 내용
    - 본문 내용 외에 주변에 표시되는 기타 정보들
    - 필수 요소가 아니므로 광고나 링크 모음 등 문서의 메인 내용에 영향을 미치지 않는 내용들을 넣을 때 사용

6. <footer></footer> 태그 : 제작 정보와 저작권 정보
    - 사이트 제작자의 연락처 정보와 저작권 정보를 표시되는

7. <address></address> 태그 : 제작자 정보와 연락처 정보
    - 사이트 제작자의 이름이나 제작자의 웹 페이지 또는 피드백을 위한 연락처 정보
    - 웹 사이트와 관련된 우편 주소 (단순 우편 주소는 <p></p> 태그 사용)

이외에도 많은 시맨틱 태그가 존재한다.





CSS 레이아웃
: 각 구성 요소 (Components)를 공간 안에 효과적으로 배치하는 프로세스

웹 레이아웃 기술 발전

1. 현재 사용 x 

    (1) 레이아웃이 없던 초창기
        - 레이아웃을 위한 디자인 방법은 별도로 없었고 제한적인 텍스트 디자인만 가능

    (2) 테이블 레이이웃
        - 테두리(border)가 투명한 테이블을 사용한 멀티 컬럼 디자인을 시작으로 웹 레이아웃이 발전
        - 하지만 테이블의 본래 목적에 위배된 방법으로 더 이상 사용되지 않음

    (3)프레임 레이아웃 
        - 테이블 레이아웃에 프레임 기술이 합쳐진 방법으로 여러 장의 HTML 문서를 결합해 하나의 문서로 사용
        - 페이지의 공통 부분을 분리, 효율적으로 관리하는데 장점이 있다.
        - 하지만 비동기 통신 기술의 발전으로 더 이상 사용되지 않음


2. 현재 사용 o

   (1). float 레이아웃 
        - HTML에서 레이아웃을 하지 않고, CSS가 제공하는 레이아웃 기술을 활용하기 시작
        - float. position 등을 사용한 방법으로 오래된 레이아웃 기술임에도 여전히 많이  사용
        - 하지만 모바일 시대에 접어들면서 멀티 스크린에 대응하는 디자인을 만들어 내기에는 부적합하다.

    (2) Flex 레아아웃
        - 다양한 모바일 환경에 대응하기 위해 등장한 레이아웃 기술
        - 유연한 박스 (Flexible box)를 사용하여 x축 또는 y축 방향으로 요소를 배치, 정렬할 수 있다.
        - 각 요소의 순서를 변경할 수 있어 기존의 레이아웃 기술 대비 높은 자유도를 제공한다.

    (3) Grid 레이아웃
        - 행/열 격자 구조에 요소를 자유롭게 배치할 수 있을 뿐더러 반응형 웹 레이아웃 대응도 가능해 각광 받는 레이아웃 기술 


3.  box-sizing 속성
    - 박스 모델 너비 값의 기준을 지정함.
    - CSS를 이용해 여러 박스 모델을 화면상에 배치하려면 박스 모델의 너비 값을 정확히 계산해야 함.
    - width 값을 계산하기 어렵다면 box-sizing: border-box 로 



(float 랑 positon 개념 더 자세히 찾아보셈 ~)
